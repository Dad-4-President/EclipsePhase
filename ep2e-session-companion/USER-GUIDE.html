User Guide
Everything you need to build, track, and maintain your operative across a campaign. No technical knowledge required.
01 Launching the app

The app runs in your browser but lives on your computer — not the internet. You start it from VS Code's terminal.

1
Open VS Code
Double-click EP2e-CharacterApp.code-workspace in your Game Tools folder. Or open VS Code and go to File → Open Workspace from File.
2
Open the terminal
Press Ctrl + ` (backtick — the key above Tab on the left). A terminal panel opens at the bottom of VS Code.
3
Type this command and press Enter
npm run dev

You'll see a message saying it's ready. Look for the line that says Local: http://localhost:5173/
4
Open your browser and go to that address
Type http://localhost:5173 in your browser's address bar and press Enter. Bookmark it for next time.
⚠️ Keep the VS Code terminal running. The app goes offline if you close VS Code or press Ctrl+C in the terminal. Just run npm run dev again to bring it back. Your character data is always safe.
02 Your first character
1
Click "New Operative" in the left sidebar
A blank character appears. The main area opens to the Ego Identity tab ready to fill in.
2
Start with a name
Type your operative's handle or callsign in the first field. This shows in the sidebar and the top bar so you can always see who's active.
3
Work through the tabs at your own pace
You don't have to fill everything at once. A good order: Identity → Aptitudes → Skills → Morph. Everything saves automatically as you type — there's no Save button.
✅ Multiple characters: Click "New Operative" again to add another. Switch between characters any time using the sidebar. Each player in your group can have their own operative in the same app.

04 Ego Identity tab

The who-you-are tab. Fill it during character creation and update it as the campaign develops.

Identity fields

Name / Aliases
Your operative's handle and any fake names or alternate identities you maintain.
Background
EP2e background from a dropdown — Colonist, Drifter, Hyperelite, etc. Used for reference against rulebook starting packages.
Career
What your character does — Hacker, Soldier, Explorer, Face, and so on.
Faction
Who your character is aligned with — Firewall, Anarchist, Hypercorp, Titanian Commonwealth, etc.
Motivations
Free text. Standard EP2e format: +Survival, -Hypercorp Control, =Uplift Rights
Rez Points
Two boxes — Unspent (available to spend) and Spent (running total of what you've used). Update after each session.

Reputation Networks

Seven rep networks scored 0–99. Update these as your GM awards or deducts rep during play. Each has a short description of what social network it covers.

Ego Traits

Click any trait button to toggle it on or off. Positive traits highlight green, negative traits highlight red. These are reference only — they don't apply mechanical penalties automatically.

Backup Record

Track when your last ego backup was made and where it's stored. In EP2e, this matters — if your morph dies, this determines what version of you is restored.

05 Aptitudes & Stats tab

Set your six aptitudes here. Everything derived — initiative, pools, thresholds — calculates automatically.

The six aptitudes

COG — Cognition
Intelligence, memory, problem-solving. Linked to most technical and knowledge skills.
INT — Intuition
Awareness, gut feeling, creativity. Drives Perceive and Research.
REF — Reflexes
Speed and reaction time. Linked to Guns, Fray, Infiltrate, and Pilot skills.
SAV — Savvy
Social awareness and emotional intelligence. Drives Deceive, Persuade, Kinesics.
SOM — Somatics
Physical fitness and body control. Linked to Athletics, Melee, and Free Fall.
WIL — Willpower
Mental fortitude and self-control. Drives Lucidity and mental health stats.

Click any aptitude value to edit it. Below each one the app shows the check value (APT × 3) — that's your base target number when rolling for that stat.

Action Pools

Insight, Moxie, Vigor, and Flex auto-calculate from your APTs. If your morph grants a pool bonus, type the override value in the box — otherwise leave it blank to use the auto value.

Damage & Wounds — tracking during play

Damage Taken
Running total of physical damage received. The health bar turns amber near Durability (unconscious) and red near Death Rating.
Wounds
Number of active wound effects. Each wound applies a –10 to all checks.
Stress Taken
Mental damage accumulated. Track against your Lucidity score.
Traumas
Active trauma effects from mental damage. Each trauma is –10 to all checks.
06 Skills tab

Two columns: Active Skills (used during play) and Know Skills (background knowledge). Skill totals auto-calculate: Rank + linked APT score.

Adding an Active Skill

1
Click "+ Add" at the top of the Active Skills column
A panel drops down with two ways to add skills.
2
Click any skill name in the quick-add list
Standard skills from the rulebook appear as buttons. Click one and it's added instantly with the correct APT already linked.
3
Set the rank
Type a number in the small "rank" box under the skill name. The Total column on the right updates immediately.
✅ Field skills like Hardware: Electronics or Medicine: Psychosurgery use the Custom section. Type the full name, pick the APT (usually COG), choose the type (technical), and set the rank.

Adding a Know Skill

Click "+ Add" in the Know Skills column on the right. Type the skill name (e.g. Know: Martian History), pick an APT, and set the rank. Know skills usually use INT or COG.

Removing a skill

Click the small ✕ on the right side of any skill row.

07 Morph / Sleeve tab

Tracks the physical body your ego currently inhabits. Update this when you resleeve.

Picking a morph

The right side of the tab shows the Morph Browser with four categories. Click the category tab, then click a morph name to select it. The stats on the left fill in automatically.

Biomorph
Standard biological bodies — Flat, Splicer, Exalt, Ghost, Fury, Olympian, and more.
Synthmorph
Robotic shells — Case, Synth, Arachnoid, Savant, Reaper.
Uplift
Uplifted animal bodies — Neo-Avian, Neo-Octopus, Neo-Gorilla, and others.
Infomorph
Pure digital existence — no physical body at all.

If your exact morph isn't listed, pick the closest one and manually edit the WT, DUR, DR, and Movement fields to match your morph's actual stats from the rulebook or your GM's notes.

Ware & Implants

The lower right panel lists all standard ware from the rulebook organized by type. Click any item to toggle it on (highlights cyan) or off. All selected ware is stored on your character record.

Armor

Select your worn armor from the dropdown. The app adds it to any armor your morph provides natively and shows a combined Energy / Kinetic total.

08 Gear & Weapons tab

Adding a weapon

1
Click "+ Add" in the Ranged or Melee section
2
Click a weapon name from the rulebook list
The weapon card appears with DV, firing mode, range, ammo, and skill already filled in.
3
Add notes if needed
Each weapon has a notes field — use it for ammo type, smart ammo loadout, mods, or anything else.

For custom or homebrew weapons, use the custom entry fields at the bottom of the add panel.

Gear Notes

The bottom of the tab is a free-text notes area for all other equipment — drugs, tools, nanofabber prints, gear packs, consumables. Write it however makes sense for your character.

09 Async / Psi tab

For Async characters only. The tab starts locked — toggle it on first.

1
Click "Not Async" to enable the panel
The button turns red and says "ASYNC ACTIVE." All the Async fields unlock below.
2
Select your Sub-Strain
Pick from Architect, Beast, Haunter, Stranger, or Xenomorph.
3
Set your Infection Rating
Type the current number. The bar changes from green (low) to amber (33+) to red (66+) matching the rulebook thresholds. Update it after any Influence event.
4
Select your Sleights
Psi-Chi sleights are passive — click to toggle on/off, they highlight purple. Psi-Gamma sleights are active — they show their infection modifier cost and highlight red when selected.

Influence Events

Six numbered fields for your sub-strain's d6 influence event table. Field 1 is pre-filled with the universal physical damage result. Fill in the rest from your sub-strain's entry in the rulebook.

10 Snapshots tab

Snapshots capture your complete character state at a moment in time — like a save point in a game. Take one at the end of every session.

Saving a snapshot

1
Update all your session changes first
Rez points spent, new skills, gear acquired, damage healed — make sure everything is current before snapshotting.
2
Type a label and click Save
Name it something descriptive like Session 3 — After resleeving or Before the Kronos run. Press Enter or click Save.

Comparing two snapshots

1
Click "Set A" on an earlier snapshot (the before)
2
Click "Set B" on a later snapshot (the after)
3
Click "Compare Snapshots"
A Delta Report appears — every change between the two points color-coded: green for new additions, amber for changed values, red for things removed.
11 How saving works
💡 Everything saves automatically. Every field you type, every dropdown you change, every skill you add — it saves to your browser instantly. There is no Save button. You can close the tab and come back later and your character is right where you left it.

Characters are stored in your browser's local storage — they survive closing the tab, closing the browser, and restarting your computer. They stay put as long as you don't clear your browser data or switch browsers.

⚠️ One risk to know about: Clearing your browser's history or cache will delete your characters. Always download a JSON backup regularly from the Snapshots tab.
12 When to take snapshots
End of each session
The most important habit. Name it with the session number and a brief note about what happened.
Before resleeving
Capture your old morph setup before switching, so you have a record of what you were running.
After spending Rez Points
Mark advancement milestones. The diff view will show exactly which skills went up.
Before a dangerous mission
A good reference point in case your character dies and you need to check their pre-mission state.
13 Backing up your character
1
Go to the Snapshots tab
2
Click "Download JSON" in the Export section
Your browser downloads a file named something like anarcat_ep2e.json. Save it somewhere safe — Documents folder, USB drive, or cloud storage.
3
To restore: click "Import Character JSON"
Paste the JSON file contents and click Import. The character reappears in the sidebar with all snapshot history intact.
✅ Download a backup JSON after every session, alongside your snapshot. Keep the last three or four — that way you can always roll back if something goes wrong.
14 Sharing with your GM

Send your GM the JSON export file via Discord, email, or however your group communicates. They import it into their own instance of the app using the Import button in the Snapshots tab.

This gives them your full character and all snapshot history — useful for tracking party advancement between story arcs or for reference during session prep.

💡 Each player runs their own copy of the app on their own machine. Characters don't sync automatically between machines, but JSON exports make it straightforward to share updates whenever needed.
15 Common questions
My characters disappeared when I opened the app today.
Browser data was probably cleared, or you opened a different browser. Always keep JSON backup exports. If you have one, use the Import button in the Snapshots tab to restore it.
The app says "localhost refused to connect" or won't load.
The dev server isn't running. Open VS Code, press Ctrl+`, and type npm run dev then press Enter. Refresh your browser once the ready message appears.
My skill totals look wrong.
Check the linked APT for that skill. Skills added from the quick-add list have the correct APT pre-set. For custom skills, make sure you selected the right APT from the dropdown. Total = Rank + APT score (Perceive and Fray use APT × 2 as the base).
The morph I want isn't in the browser.
Pick the closest morph type and manually edit the WT, DUR, DR, and Movement fields on the left side to match your morph's actual stats from the rulebook or your GM's notes.
How do I delete a character?
Click the ✕ button next to the character's name in the left sidebar. A confirmation prompt appears before anything is permanently deleted.
How do I delete a snapshot?
In the Snapshots tab, each snapshot has a small red ✕ button on the right. Click it to remove that snapshot from the history.
Can I use this on my phone?
The app runs in any browser but the layout is built for a desktop screen. It's workable on a tablet in landscape mode, but not comfortable on a phone.
I closed VS Code mid-session. Did I lose my data?
No — your character data auto-saves to the browser continuously. The only thing that stops when VS Code closes is the ability to view the app at localhost:5173. Reopen VS Code, run npm run dev, and everything is right where you left it.
